# Security Policy

## Supported versions

broask is currently maintained on the latest released version.

If you find a security issue, please report it privately before opening a public issue.

## Reporting a vulnerability

Please contact the maintainer directly through GitHub private vulnerability reporting if available, or by opening a private channel with the maintainer.

When reporting, please include:
- a clear description of the issue
- steps to reproduce it
- affected platform(s)
- impact assessment if known

Please do not publicly disclose security issues until a fix is available.
